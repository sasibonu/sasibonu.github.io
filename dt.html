Decision Tree

Decision Tree

Supervised Learning

A well-liked and adaptable machine learning technique for both classification and regression applications is the decision tree. They're simple to read and comprehend. Humans can perceive and comprehend the decision rules represented by the tree with ease. Decision Trees are made up of nodes and are hierarchical structures. Based on a feature (attribute) of the data, each node indicates a judgment. Beginning at the root node, the tree divides into branches (edges) according to the feature values. The procedure keeps going until the ultimate choice or result is represented by a leaf node. By posing a number of queries regarding the characteristics of the data, they arrive at conclusions. These inquiries are predicated on the characteristics that most effectively divide the data into several groups or classes. Maximizing the purity of the generated groups at each node is the aim.

Decision Trees are used to predict continuous numerical values in regression tasks. The average (or weighted) of the target values of the training examples that land in the same leaf node serves as the forecast for a new data point. In contrast, Decision Trees are used in classification jobs to divide data into various classes or categories. A class label is associated with each leaf node. A new data point passes through the tree by following the path that leads from the root to a leaf node; the label associated with the leaf node is the projected class label. Decision trees are frequently utilized as the fundamental components of ensemble techniques like Gradient Boosting Machines and Random Forests. Multiple Decision Trees are combined in these ensemble algorithms to enhance prediction performance and robustness.

How do I choose the correct condition to split the data?

There're many ways this can happen. Many ways to determine if the data being split is pure or impure.

1. Gini Impurity: The risk of erroneously classifying an element selected at random and labeled based on the node's label distribution is measured by the Gini impurity. A node with a Gini impurity of 0.5 represents a perfect split, where the classes are evenly distributed, whereas a node with a Gini impurity of 0 comprises instances of the same class.

2. Entropy: The degree of chaos or impurity in a set of data is measured by entropy. The uncertainty or randomness of the labels at a node can be measured using entropy in the context of Decision Trees. Label distributions that are more ordered or homogenous have lower entropies, whereas those that are more disordered or heterogeneous have higher entropies.

3. Information Gain: The reduction in entropy or impurity obtained by dividing the data based on a specific attribute is measured by information gain. It measures the amount of knowledge a feature offers regarding the class labels. When deciding which traits to split on, Decision Trees seek to maximize information gain because doing so produces more homogeneous child nodes.
